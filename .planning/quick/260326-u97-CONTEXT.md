## Task Description

Fix ESM import path issues in test files that cause `npm test` to fail.

### Problem

Tests fail with two main errors:
1. `require() is not defined in ES module scope` — tests use CommonJS `require()` syntax
2. `Cannot find module '.../bin/lib/...js'` — import paths missing `.js` extension

### Root Cause

- Test files mixed CommonJS (`require`) and ESM syntax
- TypeScript compiles to ESM but test imports don't have `.js` extensions
- Vitest configured for ESM but test files not updated

---

## Solution

### 1. Fix vitest.config.ts

Update to use vitest's native runner (not node:test):

```typescript
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,
    environment: 'node',
    // Remove setupFiles if causing issues
    include: [
      'tests/unit/**/*.test.ts',
      'tests/integration/**/*.test.ts',
      // ... keep other patterns
    ],
    // Add vitest-specific config
    passWithNoTests: false,
    testTimeout: 30000,
  },
});
```

### 2. Fix Test Import Syntax

**Pattern 1: Replace require() with import**
```typescript
// BEFORE (CommonJS)
const { test, describe, beforeEach } = require('node:test');
const path = require('path');

// AFTER (ESM)
import { describe, it, expect, beforeEach, afterEach } from 'vitest';
import * as path from 'path';
```

**Pattern 2: Add .js extension to imports**
```typescript
// BEFORE
import { ContextManager } from '../../bin/lib/context-manager';

// AFTER
import { ContextManager } from '../../bin/lib/context-manager.js';
```

### 3. Update scripts/run-tests.cjs

Ensure it uses vitest directly, not node:test runner.
